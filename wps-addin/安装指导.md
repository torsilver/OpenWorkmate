# Office Copilot — WPS 端插件安装指导

本文说明如何将 **Office Copilot WPS 加载项** 安装到 WPS 文字/表格中，以便在侧边栏与 AI 对话并操作当前文档。安装方式为**使用现成安装包 + 手工复制到本机目录**。

---

## 一、安装前准备

1. **已安装 WPS 桌面版**  
   建议使用较新版本（支持加载项功能）。若从未用过加载项，请先确认 WPS 已开启「开发人员/加载项」相关能力。

2. **已安装并配置 Chrome 扩展**  
   - Office Copilot 的 **AI 与后台地址** 均在 **Chrome 扩展** 中配置，WPS 端**不提供设置页**。  
   - 请先在浏览器中安装 Chrome 扩展，并配置好后台地址（默认 `http://localhost:8765`）与 Token。  
   - WPS 插件通过 WebSocket 连接同一后台（`ws://localhost:8765/ws?...&clientType=wps`），因此必须先保证后台已启动、扩展中 Token 正确。

---

## 二、获取安装包

本仓库已提供**可直接安装**的加载项目录：

- **路径**：本目录下的 **`office-copilot-wps_1.0.0`** 文件夹。  
- **内容**：内含 `index.html`、`main.js`、`ribbon.xml`、`taskpane.html`、`taskpane.css`、`taskpane.js` 等，无需再使用 wpsjs 构建。  
- **若你修改了任务窗格**（本目录下的 `index.html`、`taskpane.css`、`taskpane.js`），可先运行 **打包脚本** 再安装：在 `wps-addin` 目录下执行 `.\scripts\pack.ps1`（PowerShell），脚本会将上述三个文件同步到 `office-copilot-wps_1.0.0`，然后按下方步骤将整个文件夹重新复制到 jsaddons 即可完成更新。

请确认该文件夹存在且名称**必须**为 `office-copilot-wps_1.0.0`（加载项名_版本号），安装时**整个文件夹**一起复制，不要只复制其中的部分文件。

---

## 三、手工安装

### 安装路径说明（新版 WPS 用户必读）

- **不要**把插件放进 **程序安装目录**：  
  `C:\Users\你的用户名\AppData\Local\Kingsoft\WPS Office\` 下是 WPS 程序本身（含 office6、addons 等），**不要**在此处新建或放入我们的加载项。
- **正确位置**是 **用户数据目录** 下的 **jsaddons**：  
  **`%appdata%\kingsoft\wps\jsaddons`**  
  即：`C:\Users\你的用户名\AppData\Roaming\kingsoft\wps\jsaddons`  
  （Roaming，不是 Local；与「WPS Office」安装目录不同。）
- 若该路径下**没有 jsaddons 文件夹**（新版 WPS 首次安装 JS 加载项时常见），请**先新建文件夹并命名为 `jsaddons`**，再把 **`office-copilot-wps_1.0.0`** 整个文件夹复制进去。

### Windows 步骤

1. 按 `Win + R`，输入：**`%appdata%\kingsoft\wps`**，回车打开该文件夹。  
2. 若其中没有 **jsaddons**，则新建一个名为 **jsaddons** 的文件夹。  
3. 进入 **jsaddons**，将 **`office-copilot-wps_1.0.0`** 整个文件夹**复制**进来。  
   - 安装后路径示例：`%appdata%\kingsoft\wps\jsaddons\office-copilot-wps_1.0.0\`  
4. 若该目录下已有其他加载项，请勿覆盖或删除，仅新增本文件夹即可。

### Linux

- 打开目录：**`~/.local/share/Kingsoft/wps/jsaddons`**  
- 将 **`office-copilot-wps_1.0.0`** 整个文件夹复制到该目录下。

### 若加载项未出现

部分环境下，WPS 需通过 **jsplugins.xml** 注册本地加载项。若按上述步骤复制后，重启 WPS 仍看不到「Office Copilot」选项卡：

1. 在 **jsaddons** 目录下查看是否存在 **jsplugins.xml**。  
2. 若不存在或需新增本加载项，可参考安装包内的 **jsplugins.xml.example**，在 jsaddons 下编辑或创建 jsplugins.xml，加入对 `office-copilot-wps_1.0.0` 的引用（type 可为 `wps`、`et` 等，按需填写）。  
3. 具体格式与 url 写法以你当前 WPS 版本文档为准；示例中为本地 file 路径，需根据实际路径调整。

---

## 四、验证

1. **完全退出** WPS 后重新打开 **WPS 文字** 或 **WPS 表格**。  
2. 在功能区中应出现 **「Office Copilot」** 选项卡，其内有「打开 Copilot」等按钮。  
3. 点击「打开 Copilot」后，侧边栏应打开并显示本插件的聊天界面。  
4. 若 Chrome 扩展与后台已配置好，侧边栏内应显示「已连接」；若显示「未连接」，请参见下文「连接与配置说明」排查。

---

## 五、更新已安装的插件

当仓库中任务窗格有更新时：

1. 在 **wps-addin** 目录下运行打包脚本刷新可安装包：  
   **PowerShell**：`.\scripts\pack.ps1`  
   （脚本会把本目录的 `index.html`、`taskpane.css`、`taskpane.js` 同步到 `office-copilot-wps_1.0.0`。）
2. 将 **`office-copilot-wps_1.0.0`** 整个文件夹再次复制到 `%appdata%\kingsoft\wps\jsaddons`，覆盖原有同名文件夹。
3. 完全退出 WPS 后重新打开，即可使用新版本。

---

## 六、连接与配置说明

- **WebSocket 地址**：`ws://localhost:8765/ws?sessionId=xxx&token=xxx&clientType=wps`  
  - 其中 `sessionId`、`token` 由前端与后台协商（与 Chrome 扩展使用同一后台与 Token 机制）。  
- **配置入口**：所有「后台地址、AI、Token」均在 **Chrome 扩展** 中配置；WPS 端仅负责连接已配置好的后台，不在 WPS 内提供设置页。  
- 若侧栏一直显示「未连接」，请依次检查：  
  1. 本机后台服务是否已启动（默认 8765 端口）；  
  2. Chrome 扩展中配置的 Token 是否与后台一致；  
  3. 防火墙是否拦截了 `localhost:8765`。

---

## 七、进阶：自行用 wpsjs 从源码构建

若你希望从源码重新构建或修改加载项，可参考以下简要步骤：

1. 安装 Node.js 与 **wpsjs**：`npm install -g wpsjs`  
2. 使用 **wpsjs create** 创建新项目，将本目录下的 **任务窗格三件套**（`index.html` 作为任务窗格页面时可改名为 `taskpane.html`，以及 `taskpane.css`、`taskpane.js`）复制到项目的任务窗格目录并覆盖。  
3. 在项目中配置 **ribbon.xml** 与入口，使按钮点击时通过 `wps.CreateTaskpane(...)` 打开该任务窗格页面。  
4. 执行 **wpsjs build**（离线模式），将生成的 `.7z` 解压或把构建目录复制到 **jsaddons**，再按上文「手工安装」方式操作。

详细开发说明见 [WPS 加载项开发](https://qn.cache.wpscdn.cn/encs/doc/office_v11/topics/WPS%20%E5%8A%A0%E8%BD%BD%E9%A1%B9%E5%BC%80%E5%8F%91/)（若链接失效，请在 WPS 开放平台搜索「加载项开发」）。本目录功能与文件说明见 **README.md**。

---

**总结**：使用本目录中的 **`office-copilot-wps_1.0.0`** 文件夹，整体复制到 `%appdata%\kingsoft\wps\jsaddons`（Windows）或 `~/.local/share/Kingsoft/wps/jsaddons`（Linux），重启 WPS 即可完成安装；必要时配合 jsplugins.xml 注册。
