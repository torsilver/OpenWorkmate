本地服务访问密钥

建议只在此处维护一份：填写后与 AI 等配置一起点「保存配置」会写入服务端 user-config（字段 webSocketAuthToken）；若未填写 user-config，则使用 appsettings.json 中的 WebSocket:AuthToken。侧栏、Workspace 会在本机自动从引导接口同步到 chrome.storage.local；WPS/Office 任务窗格在本机首次连接时也会自动拉取同一密钥，一般无需在各端再填一遍。

保存配置后密钥会进入本机 user-config；扩展仍会把它写入 chrome.storage.local，供侧栏与 Workspace 发请求时使用。

麦克风权限（会议监听）

本选项页不能授予麦克风。麦克风由 Chrome 按站点（chrome-extension://…）单独授权，与下方 AI 等配置无关。

若提示无法访问麦克风：（1）请再次点击侧栏的「会议监听」，在浏览器弹窗中选「允许」；（2）若曾选「阻止」，打开 Chrome「设置 → 隐私与安全 → 网站设置 → 麦克风」，在列表中找到本扩展对应站点并改为「允许」；（3）也可在侧栏所在网页地址栏左侧的锁/信息图标中检查站点权限。

安全与确认（按端配置）

每个端可单独选择运行模式：RunEverything（不校验、不弹确认）、AskEverytime（每次执行前确认）、UseAllowList（仅白名单内不弹确认，名单外可确认或加入白名单）。RunEverything 会放行该端上的 CLI 命令与预定义页面脚本，请仅在完全信任环境与模型时使用。

外部 MCP 组件

安全风险

此处配置的 启动命令（Command）与参数（Args） 会由后端在本机直接启动进程，等价于在您的电脑上执行对应命令行。请勿添加不信任来源的配置。CLI 插件使用 cmd /c：在「RunEverything」模式下模型可发起任意命令；在其它模式下仍会弹出确认，但误点「允许」也会导致命令执行——请勿在不可信模型下使用 RunEverything。

配置第三方 MCP 服务器（如数据库查询工具），拓展 AI 能力。开关关闭后该组件不会随对话启动。

推荐组件

销售数据库 MCP
只读查询销售用 SQL Server；连接串在 MCP 自身 appsettings 或环境变量 SALES_DB_CONNECTION_STRING 中配置。

添加后可在下方列表中编辑「启动参数」，将项目路径改为本机路径（如 D:\CodeBase\_Taskly\sales-db-mcp\SalesDbMcp.csproj），除非后端从仓库根目录启动。

OCR MCP（图片转文字）
从图片中提取文字，供主模型整理成文档。用户附带图片发送时，对话中只会出现 attachment:xxx 引用；请先调用 File 插件的 get_attachment_path 取得本机路径，再调用 OCR 工具的 perform_ocr（或等价）传入该路径，将识别结果文本返回后由主模型组文档。

需本机已安装 Tesseract。预设使用 uvx mcp-ocr（需 Python/uv）；若用 npx 可改为 command: npx，args: -y, mcp-server-ocr（视实际包名调整）。

准确数据
将需要保证准确性的数据以文件形式存于后台目录，供 AI 写入与按 id 读取。已作为内置插件提供，无需在 MCP 列表中添加。数据目录由设置中的「准确数据目录」或配置 AccurateDataDirectory 指定。
定时任务
供 AI 创建与管理定时任务（.task.md + meta）；到点后由后台将任务内容发给 AI 执行。已作为内置插件提供，无需在 MCP 列表中添加。数据目录由设置中的「定时任务目录」或配置 ScheduledTasksDirectory 指定。
