Taskly 安装后配置（Office / Chrome / WPS）

MSI 已将程序文件安装到本目录。请先使用开始菜单中的 Start Taskly 启动本机服务（后台托盘 + Gateway + HTTPS 静态站），再按下面步骤完成各客户端一次性配置。

说明：Chrome 与 Office 商店策略限制，安装程序无法在无交互情况下自动替你「装好」扩展与加载项；下列为官方支持的旁加载 / 策略方式。

1. 启动服务

  1. 打开开始菜单 → Taskly → Start Taskly。
  2. 后台应在系统托盘出现 Office Copilot 图标；本机 HTTPS 为 https://localhost:PORT（默认端口见打包时的 StaticHost 配置，一般为 3000）。
  3. 若浏览器提示证书不受信任，请在开发环境执行 dotnet dev-certs https --trust（见主仓库 README）。

2. Microsoft Office 加载项

  1. 打开 Word 或 Excel。
  2. 插入 → 加载项 → 我的加载项 → 上传我的加载项。
  3. 选择与本目录并列的 office-addin\manifest-install.xml（完整路径即：本安装目录\office-addin\manifest-install.xml）。

3. Chrome 扩展（企业策略 / 本机更新）

  1. 确认 Start Taskly 已运行，且能访问 https://localhost:3000/taskly-chrome/updates.xml（端口以你环境为准）。
  2. 在安装目录运行 Install-ChromePolicy.ps1（右键「使用 PowerShell 运行」或在 PowerShell 中执行）。
  3. 完全退出并重新打开 Chrome，打开 chrome://policy 确认 ExtensionInstallForcelist 已生效。

4. WPS 加载项

  1. 在 WPS 中按官方文档进行「加载项 / 插件」开发或调试入口，指向本安装目录下的 wps-addin 构建产物（含 manifest.xml）。
  2. 具体菜单名称以你本机 WPS 版本为准；开发调试可使用仓库中的 wpsjs debug 流程对照。
