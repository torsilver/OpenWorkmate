# 预构建工具索引（`--build-tool-index`）使用说明

## 作用

在**不启动 Web 服务**的前提下，一次性完成：

1. **内置插件**（CLI、Word、Excel、Ppt、File、Tavily、Plan、Memory 等）的工具描述向量化，写入 `Data/rag.db`，标记 **`tool_source=builtin`**。
2. **阶段 2**：从磁盘加载 **`%LocalAppData%\OfficeCopilot\user-config.json`**（或你覆盖的配置路径），加载 **Skills 目录**下的 SKILL，并**尽力连接**配置中已启用的 **MCP**；对 **UserSkill_*** / **MCP_***（不含 STT/OCR）在各端 collection 下写入向量，标记 **`tool_source=user`**。

Embedding 调用统一使用 **`tool-index-build.json`** 中的 endpoint / apiKey / modelId（与主程序 UI 里是否已配 Embedding **无关**）。

- 改动了**内置工具**（插件函数或描述）后，请重新执行本命令以更新 builtin 索引。
- 新增或修改 **Skill / MCP** 后：可在运行时由「保存配置 / 保存技能」触发**增量**同步；若希望离线一次性对齐数据库，也可再跑本命令。

## 使用步骤

### 1. 准备配置文件

在 **backend 目录**下创建 `tool-index-build.json`（可复制 `tool-index-build.json.example` 后改名并填写真实值）：

```json
{
  "endpoint": "https://your-embedding-api/v1",
  "apiKey": "your-api-key",
  "modelId": "your-embedding-model-id"
}
```

- **endpoint**：Embedding 接口地址（必填）。
- **apiKey**：调用该接口的密钥（必填）。
- **modelId**：Embedding 模型 ID（必填）。

该文件已加入 `.gitignore`，不会提交到仓库，请勿把真实 key 提交。

### 2. 执行预构建命令

在仓库根目录或 backend 目录下执行：

```bash
dotnet run --project backend -- --build-tool-index
```

或在 backend 目录下：

```bash
dotnet run -- --build-tool-index
```

- 程序会读取 `tool-index-build.json`，写入 **backend/Data/rag.db**（若不存在会先创建 `Data` 目录）。
- 执行完成后程序直接退出，**不会启动 Web 和 WebSocket**。
- 若配置文件缺失或缺少 endpoint/apiKey/modelId 任一一项，会报错并退出码 1。

### 3. 发布时带上 DB

打包发布时，将 **backend/Data/rag.db** 一并复制到发布输出目录（例如输出目录下的 `Data/rag.db`），随后台程序一起发布。运行时若主程序使用的 RAG 存储路径与该 DB 一致（或通过配置指向该路径），即可复用已写入的向量。

## 运行时行为（与预构建分工）

- **进程启动**：**不会**批量对工具描述做 embedding；不删除、不重建工具索引。
- **用户保存配置（含 MCP）或保存/删除 Skill**：在 `RebuildKernel` 完成后，后台对 **user** 工具索引做**增量同步**：仅对新增或描述变化的条目调用 Embedding，并删除已从 Kernel 消失的 user 工具向量。
- **对话里的工具检索**：`SearchToolsAsync` 仍会对**用户当前问题**生成 query 向量（与「工具描述入库」是两件事）。

## MCP 说明

预构建**阶段 2**依赖本机能否连上各 MCP 服务；若某 MCP 未启动或连接失败，对应工具**不会**进入索引，可在服务可用后**保存一次配置**或再次运行本命令补全。

## 常见问题

- **找不到配置文件**：确保 `tool-index-build.json` 放在当前工作目录或程序运行目录（如 `backend/bin/Debug/net10.0/`）下；从仓库根目录执行时，通常需用 `--project backend`，此时「当前目录」多为仓库根目录，请把配置文件放在 backend 或运行目录下。
- **内置工具列表**：与 ChatService 中在「加载用户 Skill 和 MCP」之前注册的插件一致（如 CLI、Excel、Word、Ppt、Browser、File、System、MCP_STT、MCP_OCR、CurrentDocument、Tavily、ClawhubSkill、Memory、Context、Subagent、CrossAgentTask、Plan、UserOptions、AccurateData、ScheduledTask 等）。
- **`rag.db` 路径**：预构建默认写入**当前工作目录**下的 `Data/rag.db`；正常运行时可能使用 `%LocalAppData%\OfficeCopilot\` 下的库，请以实际配置为准，避免两处各一份导致「预构建写了但运行时读不到」。
