# 预构建工具索引（内置工具 Embedding）使用说明

## 作用

在**不启动 Web 服务**的前提下，仅对**内置插件**（CLI、Word、Excel、File、Tavily、Plan、Memory 等）做一次工具描述向量化（embedding），结果写入 `Data/rag.db`。发布时将该 DB 随程序一起打包，运行时无需每次启动都调 embedding 接口，只有用户添加/删除 Skill 或 MCP 时才会对「用户工具」做索引。

- 改动了**内置工具**（如新增/修改了某个插件的函数或描述）后，本地执行一次本命令即可重新生成内置工具索引。
- **配置与输出**：使用单独配置文件、固定输出路径，与主应用配置分离。

## 使用步骤

### 1. 准备配置文件

在 **backend 目录**下创建 `tool-index-build.json`（可复制 `tool-index-build.json.example` 后改名并填写真实值）：

```json
{
  "endpoint": "https://your-embedding-api/v1",
  "apiKey": "your-api-key",
  "modelId": "your-embedding-model-id"
}
```

- **endpoint**：Embedding 接口地址（必填）。
- **apiKey**：调用该接口的密钥（必填）。
- **modelId**：Embedding 模型 ID（必填）。

该文件已加入 `.gitignore`，不会提交到仓库，请勿把真实 key 提交。

### 2. 执行预构建命令

在仓库根目录或 backend 目录下执行：

```bash
dotnet run --project backend -- --build-tool-index
```

或在 backend 目录下：

```bash
dotnet run -- --build-tool-index
```

- 程序会读取 `tool-index-build.json`，构建仅含内置插件的 Kernel，调用 Embedding 接口生成向量，写入 **backend/Data/rag.db**（若不存在会先创建 `Data` 目录）。
- 执行完成后程序直接退出，**不会启动 Web 和 WebSocket**。
- 若配置文件缺失或缺少 endpoint/apiKey/modelId 任一一项，会报错并退出码 1。

### 3. 发布时带上 DB

打包发布时，将 **backend/Data/rag.db** 一并复制到发布输出目录（例如输出目录下的 `Data/rag.db`），随后台程序一起发布。运行时若主程序使用的 RAG 存储路径与该 DB 一致（或通过配置指向该路径），即可直接复用已写入的 builtin 向量，无需在每次启动时再对内置工具做 embedding。

## 常见问题

- **找不到配置文件**：确保 `tool-index-build.json` 放在当前工作目录或程序运行目录（如 `backend/bin/Debug/net10.0/`）下；从仓库根目录执行时，通常需用 `--project backend`，此时「当前目录」多为仓库根目录，请把配置文件放在 backend 或运行目录下。
- **内置工具列表**：与 ChatService 中在「加载用户 Skill 和 MCP」之前注册的插件一致（如 CLI、Excel、Word、Ppt、Browser、File、System、MCP_STT、MCP_OCR、CurrentDocument、Tavily、ClawhubSkill、Memory、Context、Subagent、CrossAgentTask、Plan、UserOptions、AccurateData、ScheduledTask 等）。用户自定义 Skill 和 MCP 不会在此步骤中参与索引，只会在运行时、且仅在用户变更 Skill/MCP 时做索引。
