## 默认策略（Taskly）

- **除非用户在对话中明确要求其它版式**（如全英文商务信函、创意排版、指定字体模板等），对 **中文正式书面材料**（报告、纪要、说明、对外材料等），默认按本技能与 **GB/T 9704—2012** 的常用归纳处理；不仅限于「红头文件」场景。
- 技术依据仍以国标条文为准；机关红头、公章、密级等 **专项要素** 见下文免责与「输出说明」。

## 适用范围与免责

- 本技能整理的是 **GB/T 9704—2012《党政机关公文格式》** 的常见版式要点与 **Word 实操对照**，用作 **中文正式稿默认** 的排版与自查参考，**不能替代**本单位文秘细则、红头模板或定稿审批。
- 不同地区、系统可能有 **实施细则或电子公文系统模板**，以当地规定为准。
- **正式党政公文** 另涉政治表述、文种、行文规则等，须由撰稿人与签发人把关；本技能侧重 **格式与版式**。

## 国家标准核心（版式）

依据 GB/T 9704—2012 的通行归纳（实施时请以标准全文为准）：

| 项目 | 要求摘要 |
|------|-----------|
| 纸张 | A4（210 mm×297 mm） |
| 天头 / 订口 | 上白边约 **37 mm**；左白边约 **28 mm**（版心约 156 mm×225 mm） |
| 版心 | 一般 **每面 22 行、每行 28 字**，撑满版心；文字颜色一般为 **黑色** |
| 正文 | 一般用 **3 号仿宋体** |
| 标题 | 一般用 **2 号小标宋体**，居中，多行时词意完整、对称排布 |
| 页码 | 一般用 **4 号半角宋体** 阿拉伯数字；版心下边缘之下，数字两侧各一条 **一字线**；单页码居右空一字，双页码居左空一字 |

**版头 / 主体 / 版记** 三大块：首页红色分隔线以上为版头；分隔线以下至末页首条分隔线以上为主体；末页首条分隔线以下为版记。电子稿中若无边线，仍宜按逻辑分区检查要素是否齐全。

## 结构层次序数与字体（正文内）

层次序数 **依次下行使用**，勿跳级、勿倒序：

| 层次 | 序数形式 | 字体（国标通行归纳） |
|------|-----------|----------------------|
| 第一层 | 一、 | **黑体** |
| 第二层 | （一） | **楷体** |
| 第三层 | 1. | **仿宋** |
| 第四层 | （1） | **仿宋** |

标点习惯：**一、** 后用顿号；**（一）** 后不加标点；**1.** 后用下角点；**（1）** 后不加标点。正文主体仍为 **3 号仿宋**；层次条目标题用表内字体，其下续写正文时保持仿宋与字号一致（与单位模板一致处从模板）。

## 与 `word_document_create` 配合（Taskly）

- **必须先** `load_user_skill_instructions`（`skillId` 填 **`word_cn_default_formal`**），再按本策略调用 Word 写盘；仅 `search_available_skills` **不会**把本 SKILL 正文注入模型上下文。
- **新建中文正式稿**时：请传 **`documentPreset=cnGovGbt9704`**（服务端按 GB/T 常用归纳：天头/订口近似、仿宋三号正文、固定 28 磅、标题二号宋体居中等；与 `default` 西式预设二选一）。字体依赖 **本机已安装** 仿宋/宋体等，缺失时 Word 会替换为近似字体。

### `paragraphs` 形状约定

`word_document_create` 的 **`paragraphs` 为 `string[]`（字符串数组）**：**每一项**是一段 Markdown 自然语言；多项之间由服务端按顺序用 `|` 合并后再拆段，**等价于旧版「单字符串里用 `|` 分段」**。项内仍可用空行、换行、`#` 标题、`-` 列表、或项内 `|`。

- **推荐写法**：一段落一个数组元素，例如 `["## 概述\n……", "## 要点\n一、……", "（一）……"]`；短稿也可少数几项，每项内用 `|` / 空行细分。
- **与旧习惯对照**：不要把 **整份正文** 再序列化成 **单元素** 里的 `["a","b"]` 这种 JSON 字面量字符串——应拆成多个数组元素，每项 plain Markdown。

- **易错形态（不会由服务端单独拒绝，但会得到很差的版式）**：**单个数组元素**内仍是整段 `["……","……"]` 或 curl/网页抄来的、以 `[` 开头且含大量 `\"`、`\",\"` 的转储。**即便内容来自联网检索**，也应拆成多项自然语言，勿把 API/调试输出原样塞进**某一个**元素。

- 从网页总结内容时：先提炼要点，再按「标题 + 分条」写成**多条数组元素**或「少项 + 项内 Markdown」，勿把 DOM/脚本里的结构化调试输出原样塞进 Word。

## Word 实操清单（优化时逐项核对）

1. **页面**
   - 纸张 A4；方向纵向（特殊表格页可另节横向）。
   - 页边距与国标对齐：上约 **3.7 cm**、左约 **2.8 cm**（对应天头/订口常用值）；右、下按单位模板微调，使版心接近 **22 行×28 字**（可用「页面设置 → 文档网格 → 指定行和字符网格」辅助，或固定行距见下条）。
2. **段落与行距**
   - 正文段落：**首行缩进 2 字符**；两端对齐。
   - 行距：机关单位常用 **固定值 28 磅** 与「每面 22 行」配套（若与贵单位模板不一致，以模板为准）。
3. **字体字号**
   - 标题：**2 号小标宋**、居中。
   - 正文：**3 号仿宋**。
   - 层次：按上表设置 **黑体 / 楷体 / 仿宋**，勿混用艺术字或彩色。
4. **标题与附件**
   - 多行标题注意 **词组不断开**、行距略大于正文时可接受；避免过大段前段后造成版心失控。
   - 「附件：」左空二字，附件名称后不加标点；多个附件用阿拉伯数字标序。
5. **页码**
   - 四号半角宋体阿拉伯数字；奇右偶左、外侧一字线；**公文首页是否显示页码** 按国标与单位规定（常见为首页可不加或特殊处理），勿与版记重叠。
6. **版记区**
   - 抄送机关、印发机关和印发日期等要素：分隔线、字号（常用 **4 号仿宋**）与位置以单位模板为准；**最后一个要素下加粗分隔线**。
7. **成文日期与署名**
   - 成文日期 **右空四字**；年月日用阿拉伯数字标全，不编虚位。
   - 加盖印章的版式以国标图示为准；电子版可留占位说明。
8. **清理版式**
   - 去掉多余空行顶版、隐藏直接格式、统一样式；避免用空格缩进代替首行缩进。
   - 表格、附件说明若跨页，检查 **表头重复** 与 **不与页码/版记冲突**。

## 与 Taskly 工具链的配合

- **当前文档**：优先用 `CurrentDocument` 系列读改段落、样式与节属性；路径型全文处理可用 **Word** 插件的文档级 API（与客户端能力一致）。
- **不确定版式时**：先读当前节页边距、默认字体与段落样式，再改；改后抽查 **首页、奇偶页、末页版记**。
- **长文档**：分节（封面/正文/附件）分别设页眉页脚与页码格式，避免混用同一节导致页码错乱。

## 输出给用户的说明建议

优化完成后用简短列表告知用户：**已按默认正式稿对齐的版式项**、**仍依赖单位模板或人工确认项**（如红头、公章、份号、密级、紧急程度、发文字号六角括号等），避免声称「已符合全部行政要求」。
