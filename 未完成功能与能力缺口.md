## 1. PPT：任务窗格侧「OpenXml 类」RPC（已实现；宿主差异见下）

**现状**：下列 RPC 已在 **WPS 演示**（[`useCopilot.js`](wps-addin-new/src/composables/useCopilot.js) 与 [`public/taskpane.js`](wps-addin-new/public/taskpane.js)）与 **PowerPoint**（[`office-addin/taskpane.js`](office-addin/taskpane.js)）中落地，不再统一返回「暂未实现」。

**维护约定**：修改 WPS 侧 PPT RPC 时，请运行 `node scripts/sync-public-ppt-rpc.mjs`（在 `wps-addin-new` 目录）或手动保持 `public/taskpane.js` 与 `useCopilot.js` 一致。

| RPC | WPS 演示 | PowerPoint 任务窗格 |
|-----|----------|---------------------|
| `ppt_document_create` | `Presentations.Add` + `SaveAs(filePath)` | `PowerPoint.createPresentation()`；无法按路径自动保存，需用户另存为 |
| `ppt_slide_image_add` | `Shapes.AddPicture`；可选 `imageBase64` 经临时文件 | `shapes.addPicture(base64)`；后台同机可读 `imagePath` 时会附带 `imageBase64` |
| `ppt_notes_read` / `ppt_notes_write` | `NotesPage` / `AddNotesPage` | **不支持**：Office.js 无演讲者备注 API，返回明确说明 |
| `ppt_slides_reorder` | `Slide.MoveTo` | `slide.moveTo`（零基索引） |
| `ppt_table_create` / `ppt_table_write_cells` | `Shapes.AddTable` / `Table.Cell` | `addTable` / `getTable` + `getCellOrNullObject` |
| `ppt_hyperlink_add` | `ActionSettings.Hyperlink` | `shape.setHyperlink`（需 PowerPointApi 1.10+，见 `office-addin/manifest.xml`） |
| `ppt_slide_duplicate` | `Slide.Duplicate` | `exportAsBase64` + `insertSlidesFromBase64` |

技能说明见 [`backend/Skills/ppt-pptx-1.0.0/SKILL.md`](backend/Skills/ppt-pptx-1.0.0/SKILL.md)。

## 2. WPS：删除幻灯片依赖宿主 API（按版本可能失败）

**位置**：`wps-addin-new/src/composables/useCopilot.js`（`ppt_slide_delete` 分支）

**现象**：若当前 WPS 对象模型不支持 `slide.Delete()`，会返回「当前 WPS 版本不支持删除幻灯片」，而非统一成功路径。这是**能力随宿主版本变化**的边界，不是 Chrome/OpenXml 路径的替代实现。
