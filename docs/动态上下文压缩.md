# 技术白皮书补充方案：CMA 上下文管理器触发锚点（Trigger Anchors）详解

在 Office Agent 工程中，触发锚点决定了上下文优化的**时机**与**精度**。以下详细拆解“后台自动化触发”与“主 Agent 自主调用”两种模式的逻辑设计。

---

## 一、 模式 A：后台自动化触发 (Passive/Auto Mode)
这种模式由框架（Framework）层控制，不占用主 Agent 的思考量，适合作为“系统级兜底”。

### 1. 基于 Token 窗口的物理锚点
*   **水位线触发 (Watermark Trigger)**：
    *   **低水位 (40% Context Window)**：启动“软清理”，剔除 Context 中的冗余格式代码、重复的 API 握手信息。
    *   **高水位 (70% Context Window)**：强制执行“有损压缩”，将 5 轮以前的对话全部转为简报。
*   **增量突变触发 (Delta Trigger)**：
    *   监测单次工具返回的结果（如：一次性读取了整个 Excel 工作表）。如果单词调用产生的 Token 超过单次步进阈值（如 >2000 Tokens），立即由 CMA 进行数据结构化压缩，只给主 Agent 留下提取后的统计特征。

### 2. 基于任务逻辑的状态锚点
*   **任务阶段跃迁 (Phase Transition)**：
    *   **动作锚点**：当监测到主 Agent 从 `Excel_Read` 工具切换到 `Word_Write` 或 `PPT_Export` 工具时。
    *   **逻辑**：这意味着读取阶段的大量中间变量（如单元格坐标）已失去时效。后台触发 CMA 将读取阶段的结论固化，删除过程细节。
*   **循环/报错死锁监测 (Deadlock Trigger)**：
    *   当同一工具连续报错 3 次以上，后台自动触发 CMA 进入“报错诊断模式”，清理冗余报错日志，总结失败教训并重新生成一份干净的任务现状注入主 Agent。

---

## 二、 模式 B：主 Agent 自主调用 (Active/Intent Mode)
这种模式将压缩权力交给主 Agent，通过提供名为 `optimize_memory` 或 `update_project_state` 的 **Tool (Function Calling)** 实现。

### 1. 记忆整理锚点 (Memory Organization)
*   **关键事实归档**：
    *   **触发时机**：当主 Agent 确认了一个重要的业务事实时（例如：用户确认了本次审计的基准汇率为 7.2）。
    *   **动作**：主 Agent 主动调用 CMA：“请将汇率 7.2 记入项目常驻状态，并清理之前关于汇率波动的讨论。”
*   **长文档翻页 (Context Paging)**：
    *   **触发时机**：在处理长达 50 页的 Word 文档时，主 Agent 完成了第 1-5 页的处理。
    *   **动作**：主 Agent 调用工具，要求 CMA 总结前 5 页的内容摘要，并清空前 5 页的原文 Context，以为接下来的 6-10 页腾出空间。

### 2. 认知卸载锚点 (Cognitive Offloading)
*   **复杂数据总结**：
    *   **触发时机**：当主 Agent 拿到了 5 份不同 Excel 的比对结果，感到信息量过大时。
    *   **动作**：主 Agent 显式指令 CMA：“我刚刚拿到了 5 份表格的数据流，内容太乱。请帮我生成一份结构化的差异比对表，并直接替换我的上下文。”

---

## 三、 两种模式的对比与结合建议


| 维度 | 后台自动化 (Auto) | 主 Agent 自主 (Manual) |
| :--- | :--- | :--- |
| **触发频率** | 高频率、确定性强 | 按需触发、灵活性高 |
| **主模型负荷** | 零负担（后台异步执行） | 占用 1 次推理轮次 |
| **适用场景** | 物理长度超限、常规格式清理 | 业务逻辑转折、数据深度蒸馏 |
| **实现难度** | 中（需要开发 Token 监控组件） | 高（需要高质量的 Tool-Use 指令遵循） |

### **工程落地最佳策略：双轨制**
1.  **后台监控**：作为“物理防护伞”，确保 Context 永远不会达到 100% 溢出导致崩溃。
2.  **主 Agent 显式指令**：作为“逻辑优化器”。在 System Prompt 中告诉主 Agent：“当你发现上下文中的原始文件数据干扰了你的判断时，请随时调用 `compress_data` 工具进行清理。”

---

## 四、 触发后的 Context 注入逻辑 (Memory Injection)

无论哪种模式触发，CMA 处理后的结果应按以下结构重新回填给主 Agent：

```markdown
### SYSTEM_MANAGED_STATE (不可修改)
- [Project_Root]: D:/Audit/2023_Report.xlsx
- [Key_Stats]: { "Total_Revenue": 50.2M, "Discrepancy": 0.05% }
- [Completed_Steps]: 1.数据导入; 2.异常扫描;
- [Next_Required]: 生成差异分析 PPT

### RECENT_HISTORY (保留最近 3 轮)
- User: "把异常部分标红。"
- Assistant: "已完成标红操作。"
