# 未完成功能与能力缺口（代码扫描结果）

本文档由对仓库的静态扫描与关键路径阅读整理而成，用于标注**明确未做完、与文档/工具表不一致、或仅为演示占位**的部分。  
扫描范围：`backend/`、`wps-addin-new/`、`office-addin/`、`chrome-extension/`（排除 `node_modules`、`chrome-extension/libs` 等第三方打包文件）。  
**说明**：全仓库主业务代码中几乎**没有** `TODO` / `FIXME` 注释；缺口主要来自**显式错误文案**、**RPC 分支**与**演示用占位**。

## 1. PPT：任务窗格侧「OpenXml 类」RPC（已实现；宿主差异见下）

**现状**：下列 RPC 已在 **WPS 演示**（[`useCopilot.js`](wps-addin-new/src/composables/useCopilot.js) 与 [`public/taskpane.js`](wps-addin-new/public/taskpane.js)）与 **PowerPoint**（[`office-addin/taskpane.js`](office-addin/taskpane.js)）中落地，不再统一返回「暂未实现」。后端 [`CurrentDocumentPlugin`](backend/Plugins/CurrentDocumentPlugin.cs) 对任务窗格暴露 **`current_ppt_document_create`**，经 WebSocket 调用上述 **`ppt_document_create`** RPC（与 Chrome 侧磁盘编辑用的 **`Ppt`** 插件 `ppt_document_create` 区分：后者不经任务窗格）。

**维护约定**：修改 WPS 侧 PPT RPC 时，请运行 `node scripts/sync-public-ppt-rpc.mjs`（在 `wps-addin-new` 目录）或手动保持 `public/taskpane.js` 与 `useCopilot.js` 一致。

| RPC | WPS 演示 | PowerPoint 任务窗格 |
|-----|----------|---------------------|
| `ppt_document_create` | `Presentations.Add` + `SaveAs(filePath)` | `PowerPoint.createPresentation()`；无法按路径自动保存，需用户另存为 |
| `ppt_slide_image_add` | `Shapes.AddPicture`；可选 `imageBase64` 经临时文件 | `shapes.addPicture(base64)`；后台同机可读 `imagePath` 时会附带 `imageBase64` |
| `ppt_notes_read` / `ppt_notes_write` | `NotesPage` / `AddNotesPage` | **不支持**：Office.js 无演讲者备注 API，返回明确说明 |
| `ppt_slides_reorder` | `Slide.MoveTo` | `slide.moveTo`（零基索引） |
| `ppt_table_create` / `ppt_table_write_cells` | `Shapes.AddTable` / `Table.Cell` | `addTable` / `getTable` + `getCellOrNullObject` |
| `ppt_hyperlink_add` | `ActionSettings.Hyperlink` | `shape.setHyperlink`（需 PowerPointApi 1.10+，见 `office-addin/manifest.xml`） |
| `ppt_slide_duplicate` | `Slide.Duplicate` | `exportAsBase64` + `insertSlidesFromBase64` |

技能说明见 [`backend/Skills/ppt-pptx-1.0.0/SKILL.md`](backend/Skills/ppt-pptx-1.0.0/SKILL.md)。

## 1b. WPS 文字 / 表格：CurrentDocument RPC（Vue 主线已实现）

**现状**：`excel_read_range` / `excel_write_range` / `excel_list_sheets` / `excel_get_used_range` / `excel_read_formulas` / `excel_write_formulas` 与 `word_insert_table` 已在 [`wps-addin-new/src/composables/useCopilot.js`](wps-addin-new/src/composables/useCopilot.js) 中接入，实现拆至 [`wps-addin-new/src/wps-rpc/excelRpc.js`](wps-addin-new/src/wps-rpc/excelRpc.js)、[`wordTableRpc.js`](wps-addin-new/src/wps-rpc/wordTableRpc.js)，宿主类型守卫见 [`hostKind.js`](wps-addin-new/src/wps-rpc/hostKind.js)；成功时返回的 **JSON 字符串形状** 与 [`office-addin/taskpane.js`](office-addin/taskpane.js) 中 Excel / Word 任务窗格分支一致，便于后端与模型侧无感切换。

**`public/taskpane.js`**：遗留静态栈中上述 RPC **仍为占位或未与 Vue 同步**；验收与后续修改以 **Vue + `#/taskpane`** 为准，不要求与 `public/taskpane` 双写。PPT 类 RPC 仍建议按上文第 1 节使用 `node scripts/sync-public-ppt-rpc.mjs` 与 `useCopilot.js` 保持同步。

## 2. WPS：删除幻灯片依赖宿主 API（按版本可能失败）

**位置**：`wps-addin-new/src/composables/useCopilot.js`（`ppt_slide_delete` 分支）

**现象**：若当前 WPS 对象模型不支持 `slide.Delete()`，会返回「当前 WPS 版本不支持删除幻灯片」，而非统一成功路径。这是**能力随宿主版本变化**的边界，不是 Chrome/OpenXml 路径的替代实现。

## 4. 多供应商 AI / OCR / Embedding 与语音识别（Chrome 选项页与后端字段）

**现状（已对齐）**：

- **语音识别（侧栏语音输入、会议监听、`MCP_STT` / `transcribe_audio`、`POST /api/transcribe`）**：统一依赖配置项 **`AppConfig.realtimeAsr`**（百炼实时语音识别 WebSocket v1/inference），在选项页 **「百炼实时语音识别」** 中填写 API Key、WebSocket 地址、模型 ID 等；**无**主模型 endpoint 回退、**无**独立 Whisper HTTP 整文件通道。详见 [streaming-asr-websocket-plan.md](./streaming-asr-websocket-plan.md)。
- **OCR**：支持可配置 `modelId`（百炼等）；`connectionKind` + `vendorId` 与 `OcrExtractorResolver` 一致。
- **Chrome**：[`chrome-extension/options.html`](chrome-extension/options.html) 引入 [`vendor-presets.js`](chrome-extension/vendor-presets.js)，**对话主模型 / Embedding / OCR** 等编辑区有供应商下拉；**语音识别**为单独的 `realtimeAsr` 折叠区块（不走 vendor-presets 的 STT 多列表）。保存与「测试连接」请求体携带 `vendorId`、`connectionKind`（由所选供应商推导）、OCR 的 `modelId` 等。Embedding 区块已提示更换向量模型可能导致维度不一致、需重新嵌入。

**维护**：扩展 OCR/对话等预设时在 `vendor-presets.js` 与后端 `VendorModelDefaults` / `ModelConnectionKind` 中保持同一套 id 与 `connectionKind` 字符串；语音识别以 `RealtimeAsrConfig` 与 [streaming-asr-websocket-plan.md](./streaming-asr-websocket-plan.md) 为准。

## 7. 客户端能力边界（Chrome / WPS / Office）

**配置入口**：AI 与后台相关**完整设置**主要在 **Chrome 扩展** [`chrome-extension/options.html`](chrome-extension/options.html)（`options` 页）。WPS Vue 任务窗格提供 **设置** 按钮，在配置 `VITE_CHROME_EXTENSION_ID` 时用 `window.open` 打开扩展选项页；Office（任务窗格）仍依赖已启动的 `localhost:8765` 与已保存配置，并在文案中引导到 Chrome 完成完整配置。

**仅 Chrome 扩展具备的能力**（不要求 WPS/Office 对齐）：依赖 `chrome.scripting` / `chrome.userScripts` / `tabs.captureVisibleTab` 等的**浏览器自动化**、侧栏 **Debug 面板**与 **debug-stats**、**workspace** 独立页、**语音输入**与**会议监听**（Web Speech 等）、**当前标签 `set_context`**。WPS/Office 侧以文档宿主 RPC 为主。侧栏欢迎语与 [`chrome-extension/options.html`](chrome-extension/options.html) 中有简要提示。

**WPS `public/` 目录**：任务窗格 UI 以 Vue 为准；[`wps-addin-new/public/README.md`](wps-addin-new/public/README.md) 说明 `public/taskpane.*` 仅作 RPC 同步等副产物，避免与 `TaskPane.vue` 双轨混淆。

**Pdf 插件（`get_pdf_text` / `get_pdf_info` / `pdf_document_create` / `pdf_merge`）**：在 `wps` 与 `office-*` 客户端已随 [`ClientTypeToolFilter`](backend/Services/ClientTypeToolFilter.cs) 对模型可见（`wps` 下 Pdf 与通用插件始终可见；**`CurrentDocument`** 是否收窄仅影响 Word/Excel/PPT 宿主 RPC 工具，见 `wpsHostKind`）。工具在**后端进程**上读写的路径须存在且可读/可写。若任务窗格与后端不同机或路径仅存在于用户本机而后端不可见，调用会失败——以工具返回的 `失败：…` 文案为准。与 Chrome 侧「附件落盘 + `get_attachment_path`」链路不同：`File` 插件在 **`wps` / `office-*`** 已随 `IsCommonPlugin` 暴露，但 **`get_attachment_path`** 仍主要服务 Chrome 扩展附件流；WPS 多模态/附件若需与 Chrome 对齐可另议产品方案。
