# AI 对话功能测试清单

本文档供你**与 AI 对话时手动测试功能**使用：按下面分组逐条把「用户输入」复制到对话里发送给 AI，观察是否调用预期工具、结果是否合理、错误时是否展示服务端返回的 `message`。

---

## 一、使用说明

### 1. 在哪个端测


| 端                                        | 建议测试的能力                                                                                                |
| ---------------------------------------- | ------------------------------------------------------------------------------------------------------ |
| **Chrome 侧栏**                            | Browser、File、Tavily、Memory、Plan、Subagent、CrossAgentTask、Context、AccurateData、ScheduledTask、CLI（及 HITL） |
| **Word / Excel / PowerPoint / WPS 任务窗格** | CurrentDocument（当前文档）、跨端任务「发给 Word/Excel/PPT/WPS」、计划模式与执行计划                                            |


### 2. 前置配置

- 至少配置一个可用的**主模型**（endpoint、modelId、apiKey 等）。
- 需要测的插件请提前准备好：Tavily（TAVILY_API_KEY）、Memory（Embedding）、OCR、**百炼实时语音识别**（`realtimeAsr`，测 MCP_STT / 语音相关时）、准确数据目录、定时任务目录；若在设置中配置了外部 MCP，按需准备环境等。
- 计划：计划模式在侧栏/任务窗格有入口；执行计划时先在计划页选计划并点「执行计划」，或在侧栏绑定计划后发执行类消息。

### 3. 如何记录

- 对每条提问可记录：**是否调用预期工具**、**结果是否合理**、**错误时是否展示服务端 message**。
- 对 HITL、计划确认、停止、跨端任务等，额外看：弹窗、计划页、待办列表等前端交互是否符合预期。
- 文末有**测试记录表**，可填序号、通过与否、备注。

## 二、对话测试提问列表（按功能分组）

以下每条为可直接**复制到对话里发送给 AI 的用户输入**。按顺序或按模块执行即可。

### 浏览器 (Browser)

1. 在当前网页高亮「测试」这两个字。
2. 在当前页面加一个浮动便签，标题写「提醒」，内容写「这是一条测试便签」。
3. 把当前标签页整页截图，并保存到我的下载文件夹，文件名用 `fullpage_test`。

### 文件与附件 (File) + OCR

1. （在侧栏）我附带了一张图片（先通过 UI 上传附件），请把这张图里的文字识别出来并发给我。
2. （若未配置 OCR）附带一张图，问：这张图里有什么文字？——预期应得到「需要 OCR/未配置」等可见错误说明，而不是静默失败。

### 语音转文字 (MCP_STT)

1. 我有一段录音在 `D:\temp\test.mp3`，请转成文字并总结要点。

   - 期望（按后端实现）：语音转写与侧栏语音输入、会议监听、`POST /api/transcribe` **同源**，均依赖选项页 **「百炼实时语音识别」**（`realtimeAsr`）中已填写的 API Key；未配置时应返回明确错误（如未配置百炼实时语音识别），**不会**用主模型 endpoint 回退。详见 [streaming-asr-websocket-plan.md](./streaming-asr-websocket-plan.md)。

### 命令行 (CLI) 与 HITL

1. 在 CMD 里执行 `dir` 并把结果发给我。（若为白名单且非 AskEverytime，应直接执行。）
2. 在 CMD 里执行一个通常不在白名单内的命令（例如 `whoami`；如果你已把它加入白名单则换另一个非白名单命令），确认是否弹出「需用户确认」并选择允许/拒绝，观察 AI 回复是否与选择一致。

### Excel（文件路径）

1. 在「下载」目录下创建一个新 Excel，名字叫 `test_ai.xlsx`，在 Sheet1 的 A1:B2 写入两行数据：第一行 姓名、年龄，第二行 张三、28。
2. 读取 `test_ai.xlsx` 里 Sheet1 的 A1:B2，把内容念给我听。
3. 给 `test_ai.xlsx` 的 Sheet1 的 A1 单元格设置一个 SUM 公式，比如对 A2:A10 求和。

### Word（文件路径）

1. 在下载目录创建一个新 Word 文档 `test_ai.docx`，标题写「测试文档」，正文第一段写「这是 AI 创建的段落」。
2. 读取 `test_ai.docx` 的正文前 500 字并总结。
3. 在 `test_ai.docx` 里把「测试」全部替换成「验证」。

### PPT（文件路径）

1. 在下载目录创建一个新 PPT `test_ai.pptx`，第一页标题写「封面」，第二页标题写「内容页」、正文写三条要点。
2. 读取 `test_ai.pptx` 前两页的标题和正文，用一句话概括。

### 当前文档 (CurrentDocument) — 仅任务窗格

1. （在 Word 任务窗格）在当前文档末尾插入一段话：「由 AI 通过当前文档接口插入」。
2. （在 Word 任务窗格）把当前文档正文前 300 字读给我听。
3. （在 Excel 任务窗格）列出当前工作簿所有工作表名称，并把当前表已用区域的前 5 行数据读出来。
4. （在 PPT 任务窗格）列出当前演示文稿所有幻灯片，并读出第 2 页的标题和正文。
5. （在 **WPS 演示** 任务窗格）给第 1 页写演讲者备注「验收测试」再读回；在 **PowerPoint** 任务窗格做同样请求时，应得到**明确说明**（Office.js 不支持备注），而非空成功。
6. （在 PPT 任务窗格）在第 1 页插入一张本机图片（提供同机可读路径）或先让 AI 用 Chrome+文件路径写好再切任务窗格测 `current_ppt_slide_image_add`；失败时应展示 RPC 返回的具体原因。
7. （在 PPT 任务窗格）在第 1 页添加 2×2 表格，用 `rowsCsv` 写入 `A,B|C,D`；再试非法 `newOrder`（长度不等于总页数），确认错误文案可读。

### Tavily

1. 用网络搜索查一下「Semantic Kernel 最新版本」并给我三条简要结果。
2. 从 URL `https://www.example.com` 提取正文并总结（可换成任意可访问的网页）。

### ClawhubSkill（若已配置可执行技能）

1. 运行 xxx 技能（替换为你在 Skills 里实际配置的脚本名），参数是「测试」。——确认是否调用了 `run_clawhub_script` 且结果正确或错误可见。

### 长期记忆 (Memory) — 需 Embedding

1. 请把这句话记到长期记忆里：我的偏好是报告用 Markdown 格式，不超过两页。
2. 检索一下和「报告格式偏好」相关的记忆，并告诉我有哪些。

### 准确数据 (AccurateData)

1. 往准确数据里写一条 id 为 `test-company-name` 的内容，格式 md，内容写「公司全称：测试公司」。
2. 读取 id 为 `test-company-name` 的准确数据并念给我听。
3. 列出所有 id 以 `test-` 开头的准确数据条目。
4. 删除 id 为 `test-company-name` 的准确数据。

### 定时任务 (ScheduledTask)

1. 创建一个定时任务：标题「每日提醒」，每天上午 9 点执行（cron：0 9 * * *），任务内容写「请用一句话提醒我今日待办」。
2. 列出所有定时任务，把下次执行时间告诉我。
3. 读取刚创建的那条定时任务的完整内容和 meta。
4. 把该任务的标题改成「每日 9 点待办提醒」。
5. 删除刚创建的这条定时任务。

### 上下文压缩 (Context)

1. 我们刚才聊了很多零散话题。现在开始一个新任务：请先压缩一下当前对话（把较早的轮次做成摘要），然后回答：1+1 等于几。——确认是否调用了 `compact_conversation` 且后续回答正常。

### 子代理 (Subagent)

1. 请让子代理完成这个任务：在下载目录下新建一个文本文件 `subtask_test.txt`，内容写「由子代理创建」，完成后只告诉我是否成功。——确认是否调用 `run_subtask` 且主对话只收到总结。
2. 在子代理执行过程中点击「停止」，确认是否出现「子任务已由用户停止」类提示。

### 跨端任务 (CrossAgentTask)

1. 让 Chrome 端的 Agent 帮我搜一下「今天北京天气」并回复我。（从 Word/Excel 任务窗格发起；若从 Chrome 发起可改为「让 Word 端的 Agent 在当前文档末尾插入一句话：来自 Chrome 的跨端任务」。）
2. （在目标端）打开对话，确认是否看到跨端待办并执行；执行完成后确认是否调用了 `complete_cross_agent_task` 且状态为 done。

### 计划 (Plan) — 计划模式 + 执行模式

1. 切换到「计划模式」，说：请为「在下载目录创建一个包含三页的 PPT，每页有标题和 3 条要点」制定一份实现计划。——确认是否只调用了 `create_plan`/`get_plan`、是否生成计划并返回 planId、是否需要确认（取决于步数/敏感工具配置）。
2. 在计划页打开刚生成的计划，点击「执行计划」；在侧栏确认是否绑定了该计划并自动发执行请求；确认是否按步骤调用 `execute_plan_step` 且前端 checklist 有进行中/完成状态。
3. 不切换计划模式，直接说：请执行当前选中的计划的第 1 步。——确认是否读取计划并执行对应步骤。
4. 说：把当前计划的第 2 步标记为已完成。——确认是否调用 `update_plan` 或 `complete_plan`（若支持）并反映在计划页。

### 用户技能（Prompt 型）

1. 根据你在设置里配置的某个 Prompt 型技能，提一个该技能描述能回答的问题。——确认是否调用了对应 `UserSkill_xxx` 且回复符合技能说明。

### 外部 MCP（若已在设置中配置）

1. 提一个依赖该 MCP 工具能力的具体问题（按你所接入的 MCP 文档），确认是否调用对应工具且结果或错误可见。

### 错误可见性与 API 契约

1. 故意在设置里填错主模型 API Key 或 endpoint，发一条需要调模型的普通消息，确认前端是否展示服务端返回的 `message`（而不是只显示「加载失败」）。
2. 发一条明显需要「当前文档」的消息但从 Chrome 侧栏发，确认是否得到「当前端无当前文档」类说明而非静默失败。

### 停止按钮

1. 发一条会触发多轮工具调用的请求（例如「先搜索 X，再总结并保存到准确数据」），在模型开始输出或工具执行过程中点击停止，确认流是否立即停止、界面是否有「已停止」类提示。
2. 发一条会触发子代理的请求，在子代理执行过程中点击停止，确认是否收到「子任务已由用户停止」并结束。

### 候选选项确认 (ask_options)

1. 请为我生成并让我选择两种方案（方案 A 与方案 B）：要求在侧边栏弹出候选选项确认 UI 分两步让我选择（先选方案类型，再选输出格式），不要让我在聊天里手动回复“选项编号”。当你拿到最终选择后，请把最终结果发给我。（预期：应调用 `ask_options` 并展示逐步选择 UI；最终输出基于我在侧边栏选择的内容。）
